✅ Changes We Made

  • We modernized the 2015 “A Web Game in an Hour” project using ES6+ JavaScript and modular file structure.
  • Created a total of four different site versions during the process — lots of trial and error.
  • Reorganized the original code into separate files for easier readability and debugging.
  • Upgraded the visual style from plain and dull to a more game-like, immersive design.
  • Tested the code on a temporary site to confirm everything worked before deploying it to the final website.

⚠️ Challenges We Faced

  • It was difficult to break down and understand the original code due to lack of structure and outdated syntax.
  • One bug had us stuck — the “Start Game” button wasn’t working because of a one-letter typo in the command.
  • Finals week made scheduling and progress tough — finding time to meet and collaborate was a challenge.
  • Sorting out which code belonged in which file was a slow and sometimes frustrating process.

🤝 How We Worked Together

  • We met in person twice but coordinated mostly through GroupMe to stay on track and assign tasks.
  • Framber Rodriguez focused on the core game logic, modular code structure, and bug fixes.
  • Elvis Medina handled the visual design, HTML/CSS structure, integration, and documentation.
  • We both reviewed each other’s work to make sure the project stayed clean, functional, and visually consistent.

🚧 Future Improvements

  • Adding sound effects for paddle hits and scoring
  • Introducing difficulty levels for the AI opponent
  • Implementing multiplayer mode (local or online)
  • Creating reset and pause buttons
  • Improving animations and transitions
